# DAG: Динамическое создание и наполнение таблиц

Этот проект представляет собой Airflow DAG, реализующий **динамическую ETL-пайплайн** для автоматического создания, загрузки и опциональной выгрузки данных в объектное хранилище (S3). Решение масштабируемо: достаточно добавить новую запись в конфигурацию — и DAG автоматически создаст соответствующие задачи.

---

## Основные возможности

- ✅ **Динамическая генерация задач** на основе Python-словарей.
- ✅ **Идемпотентность**: повторный запуск DAG не приводит к дублированию данных.
- ✅ **Контроль качества данных**: проверка на пустые результаты и дубликаты.
- ✅ **Автоматическое создание таблиц** при добавлении новой конфигурации.
- ✅ **Опциональная выгрузка в S3** с параметризацией бакета и пути.
- ✅ **Подробное логирование и обработка ошибок** с детализацией сбоев.


## Конфигурация (`config`)



Задачи управляются через список словарей `config`, 
где каждый элемент описывает таблицу.

---

## Логика работы DAG

1. **Старт**
2. Для каждой таблицы из `config`:
   - Создаётся таблица (если не существует) → `create_table_*`
   - Выполняется запрос на загрузку → `load_data_*`
   - Проверяется качество: нет ли пустых или дублированных данных
   - Данные вставляются после очистки таблицы (`TRUNCATE`)
   - Если `need_to_export=True`: данные экспортируются в S3 → `export_*`
3. **Завершение**

> Все задачи выполняются последовательно с зависимостями.

---

## Пример пути в S3

Если:
- `s3_bucket = 'my-data-lake'`
- `s3_key_prefix = 'exports/'`
- `table_name = 'test'`
- `ds = '2025-04-05'`

То файл будет сохранён как:
s3://my-data-lake/exports/test/2025-04-05/test.csv

## Обработка ошибок и логирование

Каждая задача содержит **расширенные логи** и **обработку исключений**:

| Сценарий | Действие |
|--------|--------|
| SQL-запрос вернул 0 строк | Ошибка + вывод самого запроса |
| Дубликаты по `id` | Ошибка + пример дублирующихся строк |
| Ошибка DDL | Ошибка + текст запроса |
| Ошибка выгрузки в S3 | Ошибка + имя таблицы и ключ |

---

## Как использовать

1. Убедитесь, что настроены `postgres_default` и `aws_default`.
2. Поместите `dynamic_table_creation.py` в папку `dags/`.
3. При необходимости добавьте новые таблицы в `config`.
4. Перезапустите Airflow (или подождите сканирование DAG).
5. Запустите DAG `dynamic_table_creation`.
