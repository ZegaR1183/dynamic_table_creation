# DAG: Динамическое создание и наполнение таблиц

Этот DAG в Apache Airflow автоматизирует создание и загрузку таблиц на основе конфигурации, загружаемой из внешнего JSON-файла. SQL-запросы хранится в отдельных файлах и включает поддержку Jinja-шаблонов для гибкости.

---

## Особенности

- **Динамическая настройка:** Позволяет задавать конфигурацию таблиц в JSON-файле, что упрощает модификации и добавление новых источников.
- **Поддержка Jinja:** Использование шаблонов Jinja в SQL-запросах для более динамичных и адаптивных операций обработки данных.
- **Контроль качества данных:** Перед загрузкой проводится проверка на дубликаты и пустые результаты, обеспечивая согласованность данных.
- **Идемпотентность:** Используется подход `TRUNCATE` для предотвращения дублирования данных при повторных запусках.
- **Выгрузка в S3:** Опциональная выгрузка данных в Amazon S3 для дальнейшего анализа или хранения, реализованная через S3Hook.
- **Легкость управления:** Структурированное хранение DDL и DML в отдельных файлах, что упрощает сопровождение и обновление.

---

## Предварительные настройки

1. **Конфигурации:** Убедитесь, что файл конфигурации таблиц `"table_configs.json"` находится по пути указанному в переменной `table_config_path`.
2. **SQL-файлы:** Проверьте, что SQL-файлы (как DDL, так и DML) находятся в директории, указанной в переменной `sql_queries_dir`.
3. **Переменные Airflow:** Настройте необходимые переменные в Airflow: `table_config_path` и `sql_queries_dir`.

---

## Логика работы DAG

1. **Старт** — задача `start_task`.
2. Для каждой таблицы из `table_configs`:
   - `create_table_<table>`: создаёт таблицу (если не существует) через DDL.
   - `load_data_<table>`:
     - Читает и рендерит DML-запрос.
     - Выполняет запрос, проверяет данные.
     - При успехе очищает таблицу и вставляет новые данные.
     - Через XCom передаёт количество строк.
   - `export_<table>` *(если `need_to_export=True`)*:
     - Экспортирует данные в CSV.
     - Загружает в S3 по шаблону пути.
3. **Завершение** — задача `end_task`.

> ✅ Все цепочки задач для таблиц выполняются **параллельно**, но внутри одной таблицы — строго последовательно.

---

## Пример пути в S3

Если:
- `s3_bucket = 'my-data-lake'`
- `s3_key_prefix = 'exports/'`
- `table_name = 'test'`
- `ds = '2025-04-05'`

То файл будет сохранён как:
s3://my-data-lake/exports/test/2025-04-05/test.csv
